# BATCH-001

Erfasst am (UTC): 2026-07-11T15:18:46Z

Kontext: Erste operative Inhalts-Batch fuer den Teacher-UI-Marschplan (Edu-Code-Core + Edu-Code-Course).

## Eingaben

### Idee 1

1. Titel: Sprachaufnahme in strukturierte Funktionsanforderungen ueberfuehren
2. Freitext oder Sprachtranskript: Lehrkraefte sprechen Ideen ein. Das System transkribiert, bereinigt Wiederholungen und zerlegt in Einzelanforderungen.
3. Prioritaet: Kritisch
4. Zielbereich: Core
5. Betroffener Prozess: Unterricht planen, Material erstellen
6. Optionaler UI-Bereich: Assistent Erstellen und Konfigurieren

### Idee 2

1. Titel: Automatische Prozess- und UI-Zuordnung
2. Freitext oder Sprachtranskript: Jede neue Idee wird vor Umsetzung auf Prozesszuordnung, Reiterzuordnung und Klickpfad geprueft.
3. Prioritaet: Hoch
4. Zielbereich: Core
5. Betroffener Prozess: Aufgabe erstellen, Aufgabe veroeffentlichen, Feedback geben
6. Optionaler UI-Bereich: Teacher UI Hauptnavigation + Unterreiter

### Idee 3

1. Titel: To-do-Generierung pro Funktionswunsch
2. Freitext oder Sprachtranskript: Aus jeder Idee entsteht automatisch eine priorisierte To-do-Liste mit Akzeptanzkriterien und technischen Teilaufgaben.
3. Prioritaet: Kritisch
4. Zielbereich: Core
5. Betroffener Prozess: Backlog-Management, Release-Planung
6. Optionaler UI-Bereich: Backlog-Panel

### Idee 4

1. Titel: Kontextbezogene KI innerhalb bestehender Arbeitsfenster
2. Freitext oder Sprachtranskript: KI-Unterstuetzung nicht nur in einem separaten KI-Reiter, sondern direkt in Kurseditor, Aufgabeneditor und Abgabenansicht.
3. Prioritaet: Hoch
4. Zielbereich: Course
5. Betroffener Prozess: Kurs verwalten, Aufgabe differenzieren, Bewertung vorbereiten
6. Optionaler UI-Bereich: Kontextaktionen in bestehenden Panels

### Idee 5

1. Titel: Milestone-Tracking per Triggerwort weiter
2. Freitext oder Sprachtranskript: Beim Stichwort weiter soll automatisch Repo-Stand, naechster Meilenstein und Testvorschlag ermittelt werden.
3. Prioritaet: Hoch
4. Zielbereich: Core
5. Betroffener Prozess: Entwicklungssteuerung, Qualitaetssicherung
6. Optionaler UI-Bereich: Systemroutine ohne separates UI

## Hinweise

- Eine Batch kann mehrere Ideen enthalten.
- Jede Idee spaeter als eigene TUI-Anforderung auspraegen.

## Batch-Ziel

Aus den 5 Ideen 5 eindeutig adressierbare TUI-Backlog-Eintraege erzeugen (TUI-001 bis TUI-005).
